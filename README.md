# 🏰 Хроники Азерота — Временная линия Warcraft

Интерактивная хронология вселенной Warcraft: **события**, **персонажи** и **зоны**
в атмосферном дизайне древней магической летописи.

Версия 2.0 — переписана на **React + TypeScript + Vite**, контент вынесен в
единый типизированный источник данных.

## ✨ Возможности

- **Таймлиния событий** с группировкой по эпохам, поиском, фильтром и липкой
  навигацией по эпохам со скролл-подсветкой.
- **Архив персонажей** с биографиями и переплетением лора: «Упоминается в
  хрониках» (события) ↔ «Действующие лица» (персонажи), всё кликабельно.
- **Хроники зон** с двумя точками зрения — Альянс и Орда.
- **Космология** — шесть фундаментальных сил мироздания (Свет/Бездна,
  Порядок/Беспорядок, Жизнь/Смерть) в трёх противоборствующих парах.
- **Спотлайт «Из хроник»** — случайное событие для начала погружения.
- **Источники с типами** (книга/игра/квест/подземелье) — иконки и ручной выбор.
- **Дип-линки** — общий доступ к событию/персонажу/зоне по прямой ссылке
  (`#/event/<id>`, `#/character/<id>`, `#/zone/<id>`).
- **Режим редактирования** прямо на сайте с экспортом/импортом JSON.
- Атмосферные анимации (framer-motion): свиток-модалка, появление карточек,
  магический фон.

## 🚀 Разработка

```bash
npm install      # установка зависимостей
npm run dev      # дев-сервер на http://localhost:5173
npm run build    # продакшн-сборка в dist/
npm run preview  # просмотр собранной версии
```

Требуется Node.js 18+ (проверено на Node 20/26).

## 📝 Наполнение контентом

Источник истины — JSON-файлы в `src/data/`. Есть два пути их пополнять:

### Через режим редактирования (рекомендуется)
1. Нажмите **«Редактировать»** в шапке.
2. Кликните по карточке для правки или **«+ Добавить»** для новой записи.
   Правки сразу сохраняются в браузере (localStorage), на сайте не теряются.
3. Когда готовы опубликовать — нажмите **«⬇ Экспорт»**, получите готовый
   `events.json` / `characters.json` / `zones.json` и положите его в `src/data/`.
4. Закоммитьте файл — после пуша в `main` сайт пересоберётся автоматически.

Кнопка **«⬆ Импорт»** загружает JSON-файл обратно в редактор, а
**«↺ Сбросить черновик»** возвращает данные к версии из репозитория.

### Правкой файлов напрямую
Можно редактировать `src/data/*.json` в любом редакторе — структура описана
в `src/data/types.ts`.

## 🎨 Дизайн

Цветовые токены — в `:root` файла `src/styles/global.css`
(арканный синий `#4fc3f7`, золото титанов `#ffd700`, фел-зелёный `#32cd32`,
фиолетовый Пустоты `#9c27b0`). Шрифты: **Cinzel** (заголовки), **Crimson Text** (текст).

## 🌐 Деплой

При пуше в `main` GitHub Actions собирает проект и публикует `dist/` на
GitHub Pages (см. `.github/workflows/deploy.yml`). Если проект лежит в
подкаталоге монорепозитория, `working-directory` в workflow уже настроен на
`WC-Timeline`. Включите Pages в настройках репозитория (Source: GitHub Actions).
