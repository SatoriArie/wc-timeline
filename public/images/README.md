# Изображения

Клади файлы сюда — пути в данные пропишутся автоматически скриптом
`node scripts/wire-images.mjs` (или попроси меня).

## Соглашение об именовании (по `id`)

| Папка | Для чего | Имя файла |
|-------|----------|-----------|
| `portraits/` | Портреты персонажей | `<id персонажа>.jpg` (напр. `arthas.jpg`) |
| `events/` | Иллюстрации событий | `<id события>.jpg` или `<id>-1.jpg`, `<id>-2.jpg` для нескольких |
| `zones/` | Виды зон | `<id зоны>.jpg` или `<id>-1.jpg`, `<id>-2.jpg` |

Поддерживаются `.jpg`, `.jpeg`, `.png`, `.webp`, `.avif`.

`id` персонажей/событий/зон — это поле `id` в соответствующем `src/data/*.json`
(латиницей, напр. `arthas`, `voyna-treh-molotov`, `tirisfalskie-lesa`).

## Как это работает

`scripts/wire-images.mjs` сканирует эти папки, сопоставляет файлы с `id` и проставляет:
- `characters[].portrait`
- `events[].images[]`
- `zones[].images[]`

Пути сохраняются относительными (`images/portraits/arthas.jpg`); в рантайме
`assetUrl()` добавляет base-путь, так что они работают и локально, и на GitHub Pages.
